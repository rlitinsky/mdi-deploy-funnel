Definitions: Licensed = active commercial tenants (gov excluded) with MDI PAU (AATPPaidAvailableUnits) > 0 and MDE PAU (MDATPPaidAvailableUnits) > 0. PAU = sum of MDI PAU; DCs = MDE-onboarded domain controllers (HeartbeatLog, 15d).

Method & sources

Scope: commercial only — gov excluded via TenantCloudType == 'Other' (removes GCC / GCC High / DoD). Licensing: MsecAllPotentialTenantsV2 (wcdprodfollower/Reporting), latest OfficeSnapshotTime (2026-07-16), active-commercial filters. Deployment / MPU / sensors: TriTelemetryEvent WorkspaceBusinessData (mdiprd/Geneva), latest per tenant over 3d. v3: ActiveDefensors (separate WBD field, not in the 5 classic counts). Auto-activation: TriTraceEvent AutoDefensorOnboardJob "Job has begun", 30d. DCs: HeartbeatLog ProductType=='DomainController' (wcdprod/Geneva), 15d, deduped by MachineId, OrgId→tenant. Node 9 MDE devices: DeviceInfoEvents.DeviceRoles (wcdprod/Geneva via ScrubbedKustoData), 15d, roles DomainController / ActiveDirectoryFederation / ActiveDirectoryCertificateServicesServer / EntraConnectServer / AzureADConnectServer.

Reconciliation: Deployed + Not-deployed = Licensed (customers, PAU, DCs — exact). v3 + v2 + no-sensor = Deployed (exact). Auto-activation ON + not-enabled = each segment (exact).

Caveats: (1) "Deployed" is MPU-gated — tenants with sensors installed but 0 MPU fall into "not deployed". (2) v3/v2 reflect currently-active sensors; a tenant whose Defensors went silent shows as v2. (3) Auto-activation = job body ran; it does not assert a successful deployment. (4) DCs = MDE-onboarded domain controllers (15d) attributed to each node's tenants; Node 9 machines are MDE identity-role devices (DC/ADFS/ADCS/EC). (5) Gov/sovereign tenants excluded.
